# Source

Application source code lives here. This directory is intentionally empty
at bootstrap — no primary language or framework has been chosen yet
(see `memory/technology-choices.md`, `docs/volumes/02-architecture.md`).

## Before Adding Code Here

1. Confirm the language/framework choice is recorded as an ADR
   (`docs/decisions/`) and in `memory/technology-choices.md`.
2. Follow `docs/standards/folder-organization.md` for internal structure.
3. Follow `docs/standards/naming-conventions.md`,
   `docs/standards/error-handling.md`, and `docs/standards/logging.md` from
   the first file onward — retrofitting standards after code exists is
   far more expensive than starting with them.
4. Mirror the structure in `tests/` per `docs/standards/testing.md`.
