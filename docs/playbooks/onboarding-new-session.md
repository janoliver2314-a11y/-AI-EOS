# Playbook: Onboarding a New Session or Contributor

Use this procedure at the start of any new Claude Code session or when a
new human contributor joins, to get to a productive, aligned starting point
quickly.

## Steps

1. Read `CLAUDE.md` in full — it is authoritative for rules and process.
2. Read `docs/volumes/01-foundation.md` for the project's mission and
   philosophy.
3. Skim `docs/volumes/10-master-reference.md` to know where to find things
   later — don't try to memorize it, just know it exists.
4. Check `memory/roadmap.md` for current priorities.
5. Before starting a specific task, check `memory/lessons-learned.md` and
   `memory/reusable-patterns.md` for anything relevant to that task's area.
6. For any non-trivial task, follow the Planning Methodology in
   `CLAUDE.md` §13 before writing code.
